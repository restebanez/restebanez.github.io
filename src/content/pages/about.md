I’m Rodrigo Estébanez, a software developer exploring how to build dependable products with language models.

My current project is [Hilvan](https://github.com/restebanez/hilvan): a system that turns fragmented email into explicit, evolving lifecycle state. It is a practical test bed for questions that appear in almost every serious LLM application:

- What should a model infer, and what should code enforce?
- How do probabilistic observations become deterministic state transitions?
- Where should provenance, confidence, and ambiguity live in the architecture?
- How can evaluations expose failures at the boundary between model and system?

This blog is an engineering notebook about those boundaries. I’ll share designs that worked, failure modes that changed the design, and reusable patterns for integrating LLMs into conventional software without pretending they are conventional functions.

You can find my work on [GitHub](https://github.com/restebanez).
